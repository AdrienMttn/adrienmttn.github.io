CGI GUEST

CONTEXT

Dans le cadre de mes deux stages au sein de CGI Clermont, j’ai eu la responsabilité de développer une application web interne destinée à gérer les badges remis aux visiteurs. Lors de mon premier stage, j’ai conçu l’application ; lors du second, j’ai apporté les correctifs et améliorations demandés par mon manager.

OBJECTIFS

L’objectif était de réaliser une application facilitant la gestion des badges remis aux visiteurs, car avant sa mise en place, la réceptionniste les gérait sur un tableau excel, ce qui compliquait la gestion et faisait peser des risques sur la traçabilité et la sécurité.

Schéma avant l’application :

[Réceptionniste]
     |
     | 1️⃣ Note le numéro du badge dans Excel
     v
[Visiteur]
     |
     | 2️⃣ Reçoit le badge
     v
[Visite terminée]
     |
     | 3️⃣ Rend le badge (à la réception ou dans la boîte)
     v
[Réceptionniste]
     |
     | 4️⃣ Marque le badge comme rendu
     v
   [FIN]

Schéma après l’application :

[Employé CGI]
     |
     | 1️⃣ Entre le nom, prénom, date d’arrivée et de départ du visiteur
     v
[Réceptionniste]
     |
     | 2️⃣ Le jour J, attribue un badge disponible via l’application
     v
[Visiteur]
     |
     | 3️⃣ Rend le badge (réception ou boîte)
     v
[Réceptionniste]
     |
     | 4️⃣ Marque le badge comme :
     |   - 🟩 Libre (si rendu)
     |   - 🟧 À désactiver (si non rendu)
     |
     v
   [FIN]

Fonctionnalités attendues :

  • Gérer les visiteurs
  • Gérer les badges
  • Créer des visiteurs
  • Gérer les rôles

COMPETENCES ET POINTS DE RÉALISATION

DIFFICULTÉS

Travailler à l’amélioration de l’application GUEST, que j’avais développée l’année précédente, m’a montré à quel point certaines décisions initiales peuvent faciliter ou, au contraire, compliquer les évolutions futures. Cette expérience m’a sensibilisé à l’importance d’un code propre, de la factorisation, de la sécurité et du choix de bonnes pratiques, afin de garantir la maintenabilité d’un projet sur le
long terme.
